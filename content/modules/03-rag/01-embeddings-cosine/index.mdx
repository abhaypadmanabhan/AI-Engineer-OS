Embed text, score similarity, plot the space. Twenty lines of numpy first, then one API call.

```python file=cosine_from_scratch.py
import numpy as np

def cosine(a: np.ndarray, b: np.ndarray) -> float:
    return float(a @ b / (np.linalg.norm(a) * np.linalg.norm(b)))

# Toy: 3 "documents" as 4-dim vectors, hand-picked so similarity is obvious
docs = np.array([
    [1.0, 0.1, 0.0, 0.0],   # cats
    [0.9, 0.2, 0.0, 0.0],   # kittens
    [0.0, 0.0, 1.0, 0.1],   # rockets
])
q = np.array([1.0, 0.0, 0.0, 0.0])  # a "cat"-like query

for i, d in enumerate(docs):
    print(f"doc{i}: cosine={cosine(q, d):.3f}")
# doc0: 0.995   doc1: 0.976   doc2: 0.000
```

That is the whole retrieval primitive. A vector, another vector, one scalar out. Everything else in Layer 3 is making those vectors *good*.

## Real embeddings in one call

Swap the hand-picked vectors for a production encoder. Voyage's free tier covers this entire layer.

```python file=voyage_embed.py
import os, numpy as np, voyageai
vo = voyageai.Client()  # uses VOYAGE_API_KEY

titles = [
    "Attention Is All You Need",
    "BERT: Pre-training of Deep Bidirectional Transformers",
    "A Simple Framework for Contrastive Learning of Visual Representations",
    "High-Resolution Image Synthesis with Latent Diffusion Models",
    "Language Models are Few-Shot Learners",
]
r = vo.embed(titles, model="voyage-3", input_type="document")
E = np.array(r.embeddings)              # shape (5, 1024)
E /= np.linalg.norm(E, axis=1, keepdims=True)   # L2-normalize once

# Dot product == cosine on normalized vectors — 2x faster than dividing every time
sims = E @ E.T
print(np.round(sims, 2))
```

<CostMeter cost="$0.00" latency="0.4s" tokens="40" />

Voyage returns 1024-dim vectors. They are *already* L2-normalized by the provider, but we normalize defensively — the next time you swap in an unnormalized encoder (fine-tuned e5, SBERT), this code stays correct.

## See the space — 2D PCA

You cannot reason about 1024 dimensions. You can reason about 2.

```python file=pca_plot.py
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt
import numpy as np, json, pathlib

rows = [json.loads(l) for l in pathlib.Path("data/arxiv_titles.jsonl").read_text().splitlines()]
titles = [r["title"] for r in rows]
cats   = [r["category"] for r in rows]

# assume `E` from the voyage call above, shape (N, 1024)
xy = PCA(n_components=2).fit_transform(E)
colors = {"cs.CL": "tab:blue", "cs.CV": "tab:orange", "cs.LG": "tab:green"}
for (x, y), c in zip(xy, cats):
    plt.scatter(x, y, c=colors[c], s=20)
plt.savefig("space.png", dpi=120)
```

Run the lab on 500 titles across three arXiv categories. Clusters separate cleanly even in 2D — that is the sanity check you want before trusting a retriever.

<Callout kind="break" title="Watch it break">
Switch `input_type="document"` to `input_type="query"` and rerun. Voyage uses *asymmetric* encoding — query and document embeddings live in slightly different subspaces. Mixing them tanks recall ~15% on real corpora. Almost every modern encoder does this; almost every tutorial gets it wrong. Check the docs before you ship.
</Callout>

## Dot vs cosine, settled

```python file=dot_vs_cosine.py
import numpy as np, time
X = np.random.randn(100_000, 1024).astype("float32")
X /= np.linalg.norm(X, axis=1, keepdims=True)
q = X[0]

t = time.perf_counter(); _ = X @ q;                                  print(f"dot:    {(time.perf_counter()-t)*1000:.1f} ms")
t = time.perf_counter(); _ = (X @ q) / (np.linalg.norm(X,axis=1)*1); print(f"cosine: {(time.perf_counter()-t)*1000:.1f} ms")
# dot: 35.2 ms   cosine: 68.4 ms  — identical result, 2x cost
```

Normalize once at ingest, dot-product at query time. That is every production vector store.
