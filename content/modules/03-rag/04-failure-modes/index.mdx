Four failure modes. No fixes yet. Learn to name them when you see them — that is the whole lesson.

```python file=reproduce_failures.py
import os
import numpy as np, cohere
co = cohere.Client(api_key=os.environ["COHERE_API_KEY"])

# Failure 1 — boundary split. A 500-char chunker cuts a table from its heading.
page = (
  "Hyperparameters\n"
  "learning_rate: 3e-4\n"
  "batch_size: 256\n"
  "warmup_steps: 2000\n"
  + "... noise " * 80
  + "Training used AdamW with cosine decay."
)
chunks = [page[i:i+80] for i in range(0, len(page), 80)]
# Query for "learning rate" retrieves the heading OR the value — never both. Signal is split.
```

### 1. Chunk boundary split

Fixed-size chunking has no respect for structure. Tables, code blocks, and numbered lists get cut. The query "what is the learning rate" returns the *heading* chunk; the query "3e-4" returns the *value* chunk. Neither is enough to answer.

### 2. Coverage gap (answered anyway)

```python file=coverage_gap.py
# Index has papers about BERT, GPT, T5. User asks about Mamba.
# Top-1 chunk is about Transformers with score 0.31 — nowhere near "relevant".
# The LLM does not see the score. It writes a plausible paragraph about Mamba that cites GPT-3.
q = "What is the recurrence relation in Mamba's state space model?"
# There is no Mamba paper in the index. The correct answer is "I don't know."
# Naive RAG never says "I don't know."
```

Coverage gap is the production RAG incident. Users trust cited answers. The citations are from papers the user didn't ask about. This costs credibility fast.

### 3. Redundancy (top-k, same doc)

```python file=redundancy.py
# Paper 2203.15556 (Chinchilla) has 12 chunks mentioning "compute-optimal".
# Query: "how was the compute budget chosen" → top-5 are all chunks 3, 4, 5, 6, 7 of 2203.15556.
# Three problems:
#   - redundant tokens burn context
#   - if the answer is in a different paper, it is now buried at rank 6
#   - the LLM over-anchors on one source
```

### 4. Numeric / exact-match queries

```python file=numeric_fail.py
# Dense embeddings are fuzzy. "73.2%" and "73.4%" and "seventy-three percent"
# all land in the same neighborhood. For exact-match queries — ticker symbols,
# error codes, config values — dense retrieval is the wrong tool.
query = "error code ECONNREFUSED in Table 3"
# Correct chunk contains the literal token "ECONNREFUSED". Top-1 is about
# "connection failures" (semantically nearest). Hybrid BM25 fixes this. Lesson 06.
```
