```python file=schema.py
from pydantic import BaseModel, Field, field_validator
from typing import Literal

class Classification(BaseModel):
    category: Literal["billing", "bug", "feature", "spam"]
    confidence: float = Field(ge=0, le=1)
    reasoning: str = Field(min_length=10, max_length=500)

    @field_validator("category", mode="before")
    @classmethod
    def normalize(cls, v: str) -> str:
        return v.strip().lower() if isinstance(v, str) else v
```

Pydantic is the only thing standing between your typed Python and the JSON-shaped lies an LLM hands back. Treat it as a firewall.

## Parse or reject

```python file=parse.py
from pydantic import ValidationError
from schema import Classification

raw = '{"category":"Bug ","confidence":0.92,"reasoning":"Stack trace cites null deref in line 42 of user.py"}'

try:
    c = Classification.model_validate_json(raw)
    print(c.category, c.confidence)     # "bug" 0.92
except ValidationError as e:
    print(e.errors())
```

Two things are happening. `model_validate_json` parses the JSON string directly in Rust, then the `normalize` validator lowers the category, then `Literal` enforces the allowed set. If any step fails you get a structured `ValidationError` with the field path.

## Reject the bad stuff on purpose

```python file=reject.py
samples = [
    '{"category":"bug","confidence":0.9,"reasoning":"real stack trace here, plenty long"}',
    '{"category":"urgent","confidence":0.9,"reasoning":"not a valid category"}',   # bad literal
    '{"category":"bug","confidence":1.4,"reasoning":"oops"}',                      # bad range + too short
    '{"category":"bug","confidence":0.9}',                                         # missing field
    'not even json',                                                               # malformed
]

def try_parse(s: str) -> Classification | None:
    try:
        return Classification.model_validate_json(s)
    except ValidationError:
        return None

parsed = [try_parse(s) for s in samples]
reject_rate = sum(p is None for p in parsed) / len(samples)
print(f"rejected {reject_rate:.0%} of garbage")   # 80%
```

A good schema rejects the bad inputs. The gate below wants ≥ 95% rejection on a pile of hand-crafted malformed payloads.

## Use it as a tool-call schema

Claude's tool use expects a JSON Schema. Pydantic gives you one for free.

```python
from schema import Classification
import json

tool = {
    "name": "classify_ticket",
    "description": "Classify a customer support ticket.",
    "input_schema": Classification.model_json_schema(),
}
print(json.dumps(tool, indent=2)[:200])
```

The Anthropic SDK will then enforce the shape server-side. You still re-validate with `model_validate` on the response — belt and suspenders, because the model can still output malformed arguments in edge cases.

## Nested models and `TypeAdapter` for lists

```python
from pydantic import TypeAdapter

batch = TypeAdapter(list[Classification])
items = batch.validate_json('[{"category":"bug","confidence":0.9,"reasoning":"xxxxxxxxxx"}]')
```

`TypeAdapter` is how you validate things that aren't top-level `BaseModel`s — lists, dicts, discriminated unions. Use it instead of the deprecated v1 `parse_obj_as`.
