Never parse a free-form LLM string in production. Force the model into a schema.

```python file=classify.py
from anthropic import Anthropic
from pydantic import BaseModel, Field
from typing import Literal

class Classification(BaseModel):
    category: Literal["billing", "bug", "feature", "spam"]
    confidence: float = Field(ge=0, le=1)
    reasoning: str

TOOL = {
    "name": "classify_ticket",
    "description": "Classify a customer support ticket.",
    "input_schema": Classification.model_json_schema(),
}

client = Anthropic()

def classify(text: str) -> Classification:
    r = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=400,
        temperature=0,
        tools=[TOOL],
        tool_choice={"type": "tool", "name": "classify_ticket"},
        messages=[{"role": "user", "content": text}],
    )
    # Tool use forces the model to emit input matching the schema
    call = next(b for b in r.content if b.type == "tool_use")
    return Classification.model_validate(call.input)

print(classify("My card was charged twice this month!"))
# category='billing' confidence=0.98 reasoning='Duplicate charge reported, billing issue.'
```

## Why this pattern wins

**Schema adherence.** The model is constrained by the tool's `input_schema`, not asked to "please return JSON." Adherence rates jump from ~92% to >99%.

**Zero regex.** You never write `re.search(r'category:\s*(\w+)', ...)` again. Pydantic parses and validates atomically.

**Typed downstream code.** The return type is `Classification`, not `dict[str, Any]`. Your IDE completes fields. Your tests mock concrete objects.

## The failure mode you must see

```python file=break.py
# Same prompt, no tool. Watch the failure variety.
r = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=400,
    messages=[{"role": "user",
               "content": "Classify: 'App crashes on login.' Return JSON."}],
)
print(r.content[0].text)
# Sometimes: {"category": "bug", ...}
# Sometimes: Sure! Here's the classification:\n```json\n{"category": "bug", ...}\n```
# Sometimes: {"Category": "Bug"} — wrong casing, crashes Pydantic
```

<Callout kind="break" title="Watch it break">
Free-form JSON output parses correctly ~9 out of 10 times on easy prompts and ~6 out of 10 on adversarial ones. That 40% failure rate is invisible in dev, catastrophic in prod. Tool use takes it to &gt;99%.
</Callout>

## Chaining structured outputs

Once outputs are typed, you can compose them without string manipulation.

```python file=chain.py
class Triage(BaseModel):
    classification: Classification
    suggested_reply: str
    escalate: bool

# pass the previous Classification object into the next tool call
# as validated JSON — the model never sees string soup
```

This is the production pattern every serious AI engineer uses. Every downstream layer — agents (L4), production services (L5), evals (every layer) — assumes every LLM call is schema-bound.
