Prompts are programs. Structure them like programs.

```python file=classify.py
import os
from anthropic import Anthropic

SYSTEM = """You are a support ticket classifier. Output only one of:
billing, bug, feature, account, other.
Do not add explanation."""

EXAMPLES = """<example>
<ticket>My card was charged twice.</ticket>
<category>billing</category>
</example>
<example>
<ticket>The app crashes when I open settings.</ticket>
<category>bug</category>
</example>
<example>
<ticket>Add dark mode please.</ticket>
<category>feature</category>
</example>"""

def classify(ticket: str) -> str:
    client = Anthropic()
    r = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=10,
        temperature=0,
        system=SYSTEM,
        messages=[{"role": "user",
                   "content": f"{EXAMPLES}\n<ticket>{ticket}</ticket>\n<category>"}],
    )
    return r.content[0].text.strip().lower()

print(classify("I cannot log in since yesterday's update."))
# account
```

## Four patterns, ranked by payoff

1. **System prompt for role + constraints.** Short. Imperative. One job.
2. **Few-shot with XML tags.** 3–5 examples that cover class boundaries, not edge cases.
3. **Structured delimiters.** `<ticket>...</ticket>` around user input so the model cannot confuse it with instructions.
4. **Explicit output format.** Tell the model what to emit and where to stop.

## Prefilling the assistant turn

Force format by starting the assistant's reply for them.

```python file=prefill.py
r = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=80,
    temperature=0,
    messages=[
        {"role": "user", "content": "List 3 arXiv IDs on RAG."},
        {"role": "assistant", "content": "["},   # prefill
    ],
)
print("[" + r.content[0].text)
# ["2005.11401", "2312.10997", "2402.01720"]
```

The model continues from `[`, so it must emit a JSON array. Cheaper than tool use for tiny one-shots.

## Delimiters matter more than you think

XML tags, triple backticks, or `---` lines all work. Mixing them confuses the model. Pick one convention per prompt and stick to it. Claude is especially trained on XML; prefer XML when structure is dense.
