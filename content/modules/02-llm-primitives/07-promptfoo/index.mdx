Prompts are code. Version them, test them, regress them.

```bash file=setup.sh
# promptfoo is an npm CLI — exception to our uv-default rule.
npm install -g promptfoo
promptfoo init
# Generates promptfooconfig.yaml + a tests folder.
```

```yaml file=promptfooconfig.yaml
description: "Support-ticket classification — 4-way prompt bakeoff"

providers:
  - id: anthropic:messages:claude-sonnet-4-6
    config:
      temperature: 0
      max_tokens: 10

prompts:
  - file://prompts/zero_shot.txt
  - file://prompts/few_shot.txt
  - file://prompts/cot.txt
  - file://prompts/cot_delim.txt

tests: file://tests/tickets.yaml

defaultTest:
  assert:
    - type: equals
      value: "{{category}}"
```

```yaml file=tests/tickets.yaml
- vars:
    ticket: "My card was charged twice this month."
    category: billing
- vars:
    ticket: "The app crashes when I open settings."
    category: bug
- vars:
    ticket: "Add dark mode please."
    category: feature
- vars:
    ticket: "I cannot log in since yesterday's update."
    category: account
# ... 10 total
```

```bash file=run.sh
promptfoo eval --output results.json
promptfoo view   # opens a local dashboard at :15500

# Cost math: 4 prompts * 10 tickets * ~200 in + ~5 out each
# input = 8000 tok, output = 200 tok
# (8000/1e6)*3 + (200/1e6)*15 = $0.024 + $0.003 = $0.027 ~ $0.05 with eval overhead
```

## Why YAML, not Python

Prompts live beside code but not in code. YAML lets you:

1. **Diff prompts in PRs.** A wording change shows up as a clean line-level diff.
2. **Branch experimental prompts.** `cot_delim.txt` is one file on a feature branch.
3. **Gate merges on regression.** CI runs `promptfoo eval --fail-on-metric accuracy<0.90`.

This is the same loop as unit tests, applied to prompts.

## Running it in CI

```yaml file=.github/workflows/prompts.yml
name: prompt-regression
on: [pull_request]
jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: {node-version: '20'}
      - run: npm install -g promptfoo
      - env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: promptfoo eval --output out.json
      - run: |
          jq '.results.stats.successes >= 36' out.json | grep true
          # 36 = 4 prompts * 10 tests * 0.9 threshold
```

<Callout kind="break" title="Watch it break">
```yaml
# A teammate opens a PR with this:
tests:
file://tests/tickets.yaml   # missing indent — now a sibling key
```
`promptfoo eval` silently loads **zero tests** because `tests` is now the empty string `""`. Every prompt "passes" vacuously. The PR merges green. Production accuracy tanks next deploy.

The fix: add a CI guard that asserts `results.stats.testCount >= N`. Also pin `promptfoo` version; new majors have changed YAML parsing rules twice in 2025.
</Callout>

## The output you actually read

```bash
$ promptfoo eval
┌─────────────┬────────┬──────────┬───────┐
│ prompt      │ pass % │ cost $   │ p95ms │
├─────────────┼────────┼──────────┼───────┤
│ zero_shot   │   0.60 │  0.004   │  820  │
│ few_shot    │   0.85 │  0.006   │  910  │
│ cot         │   0.80 │  0.018   │ 2100  │
│ cot_delim   │   0.95 │  0.019   │ 2150  │
└─────────────┴────────┴──────────┴───────┘
```

`cot_delim` wins. Commit that prompt. Next feature branch, someone edits the prompt, pass-rate drops, CI blocks the merge. That is the entire point.
