Stream responses. Never block the UI on full generation.

```python file=stream.py
import os
from anthropic import Anthropic

client = Anthropic()
with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=400,
    temperature=0,
    messages=[{"role": "user", "content": "List 5 arXiv papers on ReAct agents."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
    final = stream.get_final_message()
print(f"\n---\ninput_tok={final.usage.input_tokens} output_tok={final.usage.output_tokens}")
# Cost math: ~30 in + ~400 out => (30/1e6)*3 + (400/1e6)*15 = $0.00609 ~ $0.01
```

## OpenAI contrast

Same idea, different primitives. Keep both SDKs in your muscle memory.

```python file=openai_stream.py
from openai import OpenAI
client = OpenAI()
stream = client.responses.stream(
    model="gpt-5",
    input="List 5 arXiv papers on ReAct agents.",
    max_output_tokens=400,
)
with stream as s:
    for event in s:
        if event.type == "response.output_text.delta":
            print(event.delta, end="", flush=True)
```

The Anthropic SDK gives you a typed `text_stream` iterator plus event-level access. The OpenAI SDK exposes raw events; you filter for the delta type you want.

## Tool use over streams

Tools are the structured-output primitive (lesson 04). Streaming them needs care: the `input` JSON arrives as deltas.

```python file=tool_stream.py
TOOL = {
    "name": "get_weather",
    "description": "Look up weather for a city.",
    "input_schema": {
        "type": "object",
        "properties": {"city": {"type": "string"}},
        "required": ["city"],
    },
}

with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=200,
    temperature=0,
    tools=[TOOL],
    messages=[{"role": "user", "content": "What's the weather in Paris?"}],
) as s:
    for event in s:
        if event.type == "input_json":
            # partial_json accumulates tool.input as a string
            pass
    msg = s.get_final_message()
tool_call = next(b for b in msg.content if b.type == "tool_use")
print(tool_call.input)  # {'city': 'Paris'}
```

<Callout kind="break" title="Watch it break">
The first time you stream a tool call, you will try to `json.loads` each `input_json_delta.partial_json` chunk. Every chunk is invalid on its own — `{"ci`, `ty": "Pa`, `ris"}`. Parsing mid-stream raises `JSONDecodeError` and you ship a broken agent. The fix: ignore deltas, call `stream.get_final_message()` at `content_block_stop` and read `tool_use.input` — the SDK has already parsed it. If you must parse early for a progress UI, use a streaming JSON parser (`ijson`), not `json.loads`.
</Callout>

## Vision

One image, one line.

```python file=vision.py
import base64, httpx
img_b64 = base64.b64encode(httpx.get("https://arxiv.org/pdf/2210.03629").content[:1_000_000]).decode()
# For a real image use image/png or image/jpeg
r = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=300,
    messages=[{"role": "user", "content": [
        {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": img_b64}},
        {"type": "text", "text": "What figure is this? Cite the paper."},
    ]}],
)
print(r.content[0].text)
```

## Batch API

When latency is free and cost is not — use batches.

```python file=batch.py
batch = client.messages.batches.create(
    requests=[
        {"custom_id": f"t{i}",
         "params": {"model": "claude-sonnet-4-6", "max_tokens": 200, "temperature": 0,
                    "messages": [{"role": "user", "content": f"Classify ticket {i}: ..."}]}}
        for i in range(1000)
    ],
)
# Poll batch.id; results available within 24h at 50% discount.
```
