An LLM is a next-token predictor. That is the entire mental model.

```python file=one_token.py
import os
from anthropic import Anthropic

client = Anthropic()
r = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1,
    temperature=0,
    messages=[{"role": "user", "content": "The capital of France is"}],
)
print(r.content[0].text)
# " Paris"
```

Everything else — chat, tool use, RAG, agents — is scaffolding around repeated calls to this predictor.

```python file=loop.py
# Autoregressive in pseudocode
tokens = tokenize(prompt)
while not stop(tokens):
    logits = model.forward(tokens)          # shape: [vocab_size]
    probs  = softmax(logits / temperature)  # temperature reshapes the distribution
    next_t = sample(probs, top_p=0.95)
    tokens.append(next_t)
return detokenize(tokens)
```

Three facts that govern every API call you will ever make:

1. **Latency scales with output tokens.** Input tokens are processed in parallel. Output tokens are serial. A 2000-token prompt with a 100-token answer is faster than a 100-token prompt with a 2000-token answer.
2. **Cost scales with both, asymmetrically.** Output tokens are 3–5x the price of input tokens on frontier models. Prefer retrieval + short outputs over long generated context.
3. **`temperature=0` is a knob, not a promise.** Providers do not guarantee byte-identical output across runs. Tests that compare raw strings will flake.
