Tokens are the currency. Measure before you spend.

```python file=count.py
from anthropic import Anthropic

client = Anthropic()
prompt = "Summarize the attached SEC 10-K filing in 3 bullets."
r = client.messages.count_tokens(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": prompt}],
)
print(r.input_tokens)  # 14

# Cost math (claude-sonnet-4-6, $3/$15 per Mtok):
# 14 input tokens -> (14/1_000_000) * 3 = $0.000042
# Plan 300 output -> (300/1_000_000) * 15 = $0.0045
# Total per call ~ $0.0045. 1000 calls ~ $4.50.
```

## Tokenization, concretely

Tokens are sub-word pieces. "engineering" is one token; "refactoring" is two. Whitespace sticks to the following word: `" Paris"` is one token, `"Paris"` is another. For GPT models, use `tiktoken`:

```python file=tiktoken_demo.py
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4o")  # closest proxy for gpt-5
ids = enc.encode("def classify_ticket(text: str) -> Ticket:")
print(len(ids), ids[:5])  # 12 [755, 67, 5127, 18183, ...]
```

Rule of thumb: **1 token ≈ 4 English characters ≈ 0.75 words**. Code is denser — assume 1 token ≈ 3 chars. Non-English can be 2x.

## Context windows are not free RAM

Claude Sonnet 4.6 accepts 200k input tokens. That does not mean you should send 200k tokens.

```python file=cost_at_scale.py
# 200k input, 500 output, sonnet-4-6
in_cost  = (200_000 / 1_000_000) * 3   # $0.60
out_cost = (500     / 1_000_000) * 15  # $0.0075
print(f"one call: ${in_cost + out_cost:.4f}")  # $0.6075
# 10k users x 10 calls/day = $60k/day. Do the math before shipping.
```

Long contexts also suffer the *lost-in-the-middle* effect: retrieval accuracy dips for facts placed around token 100k. Either shrink the context (RAG, Layer 3) or put the important span first or last.

## Temperature and top_p

`temperature` rescales logits before softmax. `top_p` truncates the distribution to the smallest set whose cumulative probability ≥ p.

```python file=temp.py
import os
from anthropic import Anthropic
client = Anthropic()

def joke(temperature: float) -> str:
    r = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=40,
        temperature=temperature,
        messages=[{"role": "user", "content": "A one-line dev joke."}],
    )
    return r.content[0].text

print(joke(0.0))  # near-identical across runs
print(joke(1.0))  # varies wildly
```

Defaults for production: `temperature=0` for any path with a test, `temperature=0.7` only for copy generation. Do not touch `top_p` unless you have a measured reason.
