## Loading keys the right way

```python file=load_env.py
# Never hardcode. Never commit. Always load from env.
import os
from dotenv import load_dotenv

load_dotenv()  # reads .env in cwd, no-op in prod (env already set)

ANTHROPIC_KEY = os.environ["ANTHROPIC_API_KEY"]   # raises if missing — good
OPENAI_KEY    = os.environ.get("OPENAI_API_KEY")  # None if missing — only for optional
```

<Callout kind="warn" title="Never do this">
```python
# DO NOT
client = Anthropic(api_key="sk-ant-abc123...")
```
A single commit with a hardcoded key will leak. Anthropic auto-rotates pushed keys within minutes, but attackers can still burn through your credit in the window before detection.
</Callout>

## Per-project isolation with direnv

```bash file=.envrc
# .envrc is committed, .env is not
dotenv
export ANTHROPIC_LOG=info            # per-project only
```

When you `cd` out of the project, `direnv` unloads the vars. No key pollution across projects.

## Verify in 5 seconds

```python file=verify.py
import os
from dotenv import load_dotenv

load_dotenv()

required = ["ANTHROPIC_API_KEY", "OPENAI_API_KEY", "HUGGINGFACE_TOKEN"]
missing = [k for k in required if not os.environ.get(k)]
assert not missing, f"missing keys: {missing}"
print("OK — all keys loaded")
```
